# 黑客松大学 (Hackathon University) 网页应用设计规范

> **基于 AttraX 品牌视觉体系 × 产品 PRD v0.1**
> 版本：v2.0 · 2026年4月

---

## 一、 产品愿景与设计哲学

### 1.1 产品定位

根据 PRD，"黑客松大学"不是单纯的活动信息平台，而是一个围绕**发现层（信息）→ 沉淀层（资产）→ 认证层（标准）**三层结构建立起来的创造者档案系统。它的核心目标是：让全球的 Outlier 在这里发生连接，把参赛经历沉淀为长期可信的个人资产。

> *"这个时代，平等惩罚每个平庸的人。明星员工、生态贡献者，价值在流动。"*
> — 产品讨论记录

### 1.2 设计哲学

AttraX 的品牌底色是**「快乐、叛逆、自由」**，品牌宣言是 **"We engineer the UNNECESSARY"**。这意味着"黑客松大学"的设计必须打破常规 SaaS 平台的沉闷感，创造一个**赛博朋克风格的地下车库**——粗糙、真实、充满未知的能量。

设计的核心张力来自两组对立的融合：**极度精准的工程感 × 随机爆发的创造力**。

### 1.3 三大设计原则

| 原则 | 说明 | 对应 PRD 场景 |
| :--- | :--- | :--- |
| **数据极客感** | 等宽字体、网格线、终端 UI、数字可视化 | 热力图、足迹地图、氛围指标、徽章认证状态 |
| **叛逆破局感** | 大胆排版、不对称布局、手写涂鸦 | 首页 Hero、GOAT Hunt 排行榜、个人档案 |
| **真实粗犷感** | 硬边框、纯色块、拒绝过度修饰 | 所有卡片组件、按钮、分割线 |

## 二、 全局色彩系统 (Color System)

全站采用**极高对比度黑白主调 + 单一高饱和强调色**的深色模式（Dark Mode）。

│  Pure Black  │  Deep Black  │  Card BG     │  Elevated Surface   │
│  主背景      │  次级背景    │  卡片背景    │  悬浮层/模态框      │
└──────────────┴──────────────┴──────────────┴─────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│  NEUTRAL — 中性灰色系                                             │
├──────────────┬──────────────┬──────────────┬─────────────────────┤

### 2.2 强调色板

强调色来源于 AttraX 视觉素材的像素级色彩提取，是品牌的灵魂所在。

```
┌──────────────────────────────────────────────────────────────────┐
│  ACCENT — 强调色系（仅用于关键交互与视觉焦点，占比 ≤ 5%）         │
├──────────────┬──────────────┬──────────────┬─────────────────────┤
│  #F56B52     │  #D94F38     │  #F5C842     │  #00FF41            │
│  AttraX      │  AttraX      │  Amber       │  Terminal Green     │
│  Coral Red   │  Coral Dark  │  琥珀黄      │  终端绿             │
│  主强调色    │  按压/深色态 │  热度/待审核 │  认证成功/上线      │
└──────────────┴──────────────┴──────────────┴─────────────────────┘
```

**色彩来源说明**：珊瑚橙红 `#F56B52` 直接提取自九坤合作图中"outlier"手写字与标题文字的橙红色笔触，是 AttraX 最核心的品牌强调色；琥珀黄 `#F5C842` 作为暖色系配套强调色，用于热度、待审核等次级语义；终端绿 `#00FF41` 是黑客文化的经典语义色，用于 `verify_status = verified` 等成功状态。

### 2.3 语义色映射

| 业务语义 | 颜色 | 对应数据字段 |
| :--- | :--- | :--- |
| 已认证 (Verified) | `#00FF41` 终端绿 | `verify_status = verified` |
| 待审核 (Pending) | `#F5C842` 琥珀黄 | `verify_status = pending` |
| 已拒绝 (Rejected) | `#606060` 灰色 + 删除线 | `verify_status = rejected` |
| 报名中 (Open) | `#F56B52` 珊瑚橙红 | `registration_status = 报名中` |
| 已结束 (Closed) | `#606060` 灰色 | `registration_status = 已结束` |
| 热度/点赞 (Hot) | `#F56B52` 珊瑚橙红 | `like_count`, `rank_score` |

## 三、 字体排版系统 (Typography)

### 3.1 字体家族

| 用途 | 字体 | 引入方式 |
| :--- | :--- | :--- |
| **英文大标题 / Hero** | [Space Grotesk](https://fonts.google.com/specimen/Space+Grotesk) Black | Google Fonts |
| **中文大标题** | 思源黑体 Heavy / 阿里巴巴普惠体 Heavy | CDN 引入 |
| **正文 (中英)** | [Inter](https://fonts.google.com/specimen/Inter) Regular/Medium | Google Fonts |
| **数据/代码/标签** | [JetBrains Mono](https://www.jetbrains.com/lp/mono/) Regular | Google Fonts |
| **装饰 Slogan** | [Caveat](https://fonts.google.com/specimen/Caveat) Bold | Google Fonts |

### 3.2 字体比例 (Type Scale)

```
Display  — 80px / Space Grotesk Black / ALL CAPS / letter-spacing: -0.02em
H1       — 48px / Space Grotesk Bold
H2       — 32px / Space Grotesk Bold
H3       — 24px / Space Grotesk Medium
Body     — 16px / Inter Regular / line-height: 1.75
Small    — 13px / Inter Regular / color: #A0A0A0
Mono     — 13px / JetBrains Mono / color: #A0A0A0
Caption  — 11px / JetBrains Mono / ALL CAPS / letter-spacing: 0.1em
```

### 3.3 关键排版规则

**超大标题 (Hero Typography)**：首页 Slogan 使用 Display 级别字体，跨越两行，每行一个词，充满整个屏幕宽度，参考海报中"BUILD THE NEXT"的处理方式。

**等宽字体混排**：在中文语境中，用户 ID、时间戳、黑客松等级分数（`level_score`）、标签（`tags_json`）等数据字段一律使用 JetBrains Mono，在视觉上形成"数据"与"叙事"的层次区分。

## 四、 视觉元素与图标 (Visual Elements & Iconography)

### 4.1 图标规范

*   **图标库**：推荐使用 [Phosphor Icons](https://phosphoricons.com/)（线性风格，支持 React/Vue）。
*   **线条粗细**：统一 1.5px 或 2px，不使用填充（Solid）图标。
*   **圆角**：图标本身无圆角，与整体硬朗风格一致。

### 4.2 品牌视觉符号 (Brand Motifs)

以下元素直接来源于 AttraX 视觉素材，可作为网页背景装饰、分隔符或插图：

**网格坐标 (Engineering Grid)**：细灰线（`#1E1E1E` 至 `#2A2A2A`）构成的背景网格，透明度 20%-30%，用于首页 3D 地球背景、个人档案页背景。

**星图与拓扑网络 (Constellation Network)**：节点与连线构成的图形，象征全球 Outlier 的连接。适用于首页"全球创造者星图"的视觉概念，以及社区/团队页面的背景插图。

**复古数字界面 (Retro-Digital)**：波形图、频谱图、二进制字符流等元素，用于 Loading 动画、空状态（Empty State）或 Hero 区域的装饰性背景。

**荒诞拼贴 (Absurdist Collage)**：龙虾与宇宙星空、飞马与星图的混搭风格，用于 404 页面、活动宣传 Banner 或空状态插图。

## 五、 核心模块页面设计指引 (Page Design Guidelines)

### 5.1 首页展示模块 (Home)

**视觉中心：3D 创造者星图**

首页的核心是一颗基于 WebGL（如 Three.js）渲染的黑色/暗橙色 3D 地球，表面布满发光的节点（代表 Hackathon 地点，来自 `hackathon.latitude` 和 `hackathon.longitude`）。节点颜色根据 `registration_status` 区分：报名中为珊瑚橙红，已结束为灰色。

悬停节点时弹出的信息卡片应具有硬边框（`border: 1px solid #F56B52`）和等宽字体，展示 `title`、`start_time`、`city`、`tags_json` 和 `level_score`。

**氛围指标面板 (Vibe Metrics)**

在页面侧边或地球下方，使用终端风格展示 `stat_snapshot` 中的数据：

```
> BUILDERS CONNECTED:  3,021
> PROJECTS SHIPPED:    1,204
> CITIES COVERED:        47
> BADGES EARNED:         892
█
```

数字应有滚动计数动效，末尾有闪烁的光标 `█`。

**内容流 (Featured & Trending)**

地球下方采用不对称两栏布局：左侧 60% 展示 Featured Hackathons，右侧 40% 展示 Trending Projects（按 `like_count` 排序）。

---

### 5.2 黑客松模块 (Hackathons)

**列表页 (Hackathon List)**

卡片采用 `border-radius: 0` 的硬边框设计。左侧是海报缩略图（默认黑白处理，Hover 时恢复彩色），右侧是结构化信息。标签（`tags_json`）使用等宽字体呈现为 `[#AI]` `[#Hardware]` 的代码注释风格。

`level_score` 以等宽字体数字形式展示，旁边附带 `level_code` 的文字说明（如 `S级 · 98.5`）。

**详情页 (Hackathon Detail)**

Hero 区域使用全宽海报作为背景，叠加极粗的 Space Grotesk 标题。页面主体采用类似 GitHub Readme 的排版风格展示 `long_desc`，保留 Markdown 的原生粗犷感。

参与该活动的用户卡片（来自 `user_hackathon_record`）以横向滚动列表展示，每张卡片显示头像、`display_name` 和 `role_type`。

---

### 5.3 News / Blog 内容模块 (Stories)

**列表页 (Story List)**

类似 Medium 的极简排版，但背景为纯黑，文字为高对比白。卡片左侧是缩略图（`cover_url`，无则使用默认兜底图），右侧是标题、摘要、作者头像和发布时间。

标签（`tags_json`，如 案例/访谈/思考）使用等宽字体，与黑客松标签风格保持一致。

**详情页 (Story Detail)**

正文内容（`content_md` 或 `content_html`）使用专为深色模式优化的 Markdown 渲染样式：代码块使用终端绿 `#00FF41` 高亮，引用块使用珊瑚橙红左边框。

---

### 5.4 GOAT Hunt 作品模块 (Projects)

**列表页 (Project Leaderboard)**

类似 Product Hunt，但视觉更 Hacker。每行左侧是巨大的排名数字（`rank_score`，使用 Space Grotesk Black，字号 48px），右侧是作品信息。

点赞按鈕（Upvote）设计成类似街机游戏投币的按鈕，点击时数字跳动，颜色变为珊瑚橙红 `#F56B52`，同时触发轻微的震动动效。

**作品详情页 (Project Detail)**

强调 `github_url` 和 `demo_url` 两个核心按鈕，使用主按鈕样式（白底黑字，Hover 变珊瑚橙红）。团队成员（`work_member` 或 `team_member_text`）展示为一排带有 `@` 前缀的等宽字体 ID 链接。

所获奖项（`award_text`，`is_awarded`）展示为珊瑚橙红背景的徽标，字体全大写。

---

### 5.5 用户档案模块 (Geek Profile)

用户档案页是平台最核心的资产沉淀页面，设计上要像一份**硬核的技术履历**。

**头部 (Profile Header)**

头像使用八边形裁切（`clip-path: polygon(...)`），拒绝正圆形，体现 Outlier 的不规则感。`display_name` 使用 H1 级别字体，`bio` 使用正文字体。`looking_for` 状态（如 cofounder, teammate）以珊瑚橙红边框的标签展示。

**核心看板 (Stats Dashboard)**

四个巨大的数字块，展示 `badge_count`、`total_hackathon_count`、`total_award_count`、`total_work_count`。数字使用 Space Grotesk Black，字号 48px，标签使用 Caption 级别等宽字体（ALL CAPS）。

**行动视图 (Action Views)**

三种视图通过 Tab 切换，Tab 标签使用等宽字体：

*   **足迹视图 (FOOTPRINTS)**：基于 `user_hackathon_record` 中的 `city` 和 `country` 数据，在极简 2D 世界地图上绘制参赛轨迹连线，类似"航旅纵横"的飞行路线。连线颜色为珊瑚橙红 `#F56B52`，起止点为发光圆点。底部展示 `总覆盖城市数 / 总覆盖国家数`。

*   **热力图 (HEATMAP)**：基于 `user_activity_log` 中的 `activity_date` 渲染类似 GitHub 贡献日历的热力图。颜色从 `#1A1A1A`（无活动）渐变到 `#F56B52`（高活跃），参加黑客松的日期使用最高亮的珊瑚橙红。

*   **结构化表格 (RECORDS)**：基于 `user_hackathon_record` 的全部字段，使用等宽字体渲染类似终端输出的数据表格，字段包括黑客松名、时间、地点、`role_type`、`rank_text`、`verify_status`。

**徽章墙 (Badge Wall)**

基于 `user_badge_record` 展示。已点亮的徽章（`verify_status = verified`）使用全彩显示，未获得的徽章置灰（透明度 30%）。徽章图标设计应采用**复古像素风或 3D 金属质感**，与整体极客风格呼应。

---

### 5.6 认证与徽章模块 (Verify & Badges)

**徽章中心 (Badge Center)**

展示全部 `badge_definition` 中的徽章，按 `badge_type`（award/participation/milestone/community）分类。每个徽章卡片展示图标、名称、`rule_desc`（获取规则）和已获得人数。

**认证状态设计**

`verify_status` 是该模块最核心的视觉元素，四种状态的视觉设计如下：

| 状态 | 颜色 | 边框样式 | 附加效果 |
| :--- | :--- | :--- | :--- |
| `pending` 待审核 | `#F0C040` 琥珀黄 | 1px 虚线 (dashed) | 闪烁动效 |
| `verified` 已认证 | `#00FF41` 终端绿 | 1px 实线 | 勾号图标 |
| `rejected` 已拒绝 | `#606060` 灰色 | 1px 实线 | 删除线 |
| `revoked` 已撤销 | `#333333` 深灰 | 1px 虚线 | 感叹号图标 |

## 七、 布局与间距 (Layout & Spacing)

### 7.1 网格系统

全站采用 12 列网格，最大内容宽度 `1280px`，内边距 `24px`（移动端）/ `48px`（桌面端）。

### 7.2 间距单位 (8px Scale)

| 变量名 | 数值 | 典型用途 |
| :--- | :--- | :--- |
| `--sp-1` | 4px | 图标与文字间距 |
| `--sp-2` | 8px | 标签内边距 |
| `--sp-3` | 16px | 卡片内元素间距 |
| `--sp-4` | 24px | 卡片内边距 |
| `--sp-5` | 32px | 组件间距 |
| `--sp-6` | 48px | 章节间距 |
| `--sp-8` | 64px | 大章节间距 |

### 7.3 不对称布局 (Asymmetric Layout)

在首页 Hero 区、用户档案页等关键区域，使用不对称两栏布局（如 65% / 35%），形成视觉张力，体现 Outlier 不规则的精神。

## 八、 动效规范 (Motion & Animation)

动效应体现"代码运行"的感觉，而非流畅的消费级 App 动画。

| 场景 | 动效类型 | 时长 | 缓动函数 |
| :--- | :--- | :--- | :--- |
| 卡片悬停 | 位移 2px + 边框变色 + 硬阴影 | 150ms | `ease` |
| 按钮点击 | 轻微压缩 `scale(0.97)` | 100ms | `ease` |
| 点赞触发 | 数字跳动 + 颜色变红 | 200ms | `ease-out` |
| 氛围指标数字 | 终端数字滚动计数 | 1000ms | `linear` |
| 加载状态 | 光标 `█` 闪烁 | 循环 | `step-end` |
| 待审核徽章 | 虚线边框闪烁 | 循环 1.5s | `step-end` |
| 热力图格子 | 渐入 (fade-in) | 500ms | `ease-out` |

## 十、 后台运营模块设计建议 (Admin Panel)

根据 PRD，后台主要面向官方运营（审核、配置 Banner、查看数据统计），设计原则是**工具属性拉满，极度克制**。

全盘采用黑白灰 + 终端绿（正常状态）/ 龙虾红（异常/拒绝状态），不使用花哨的紫色装饰。数据看板直接使用原生表格和最基础的折线图展示 `stat_snapshot` 数据，追求最高的信息密度。

`verification_apply` 的审核界面采用左右分栏：左侧展示用户提交的 `proof_url`（图片/链接），右侧展示对比信息与 Approve / Reject 按钮，支持快捷键操作（`Y` 通过，`N` 拒绝），以提升审核效率。

## 十一、 设计禁忌 (Anti-Patterns)

| 禁止 | 原因 |
| :--- | :--- |
| 白色/浅色背景作为主色调 | 违背 AttraX 极客黑的品牌基调 |
| 过度圆角（> 8px） | 显得过于"消费级"，失去叛逆感 |
| 柔和的渐变色背景 | 与品牌的硬核、直接风格相悖 |
| 大量使用 Emoji 作为图标 | 降低专业感，应使用线性图标 |
| 绝对对称的网格布局 | 过于规则，违背 Outlier 精神 |
| 蓝色/紫色系作为主强调色 | 与 AttraX 珊瑚橙红/琥珀黄品牌色冲突 |
| 正圆形头像裁切 | 应使用方形或八边形，体现不规则感 |
